Sync pipeline every 15 min

Entry point: scripts/sync_now.bat → python -m scripts.sync_now

1Ingest
full_scan + scan_onedrive → processor.process_file: hash → dedup → parse filename → classify → upload GCS → vendor match → upsert. On duplicate SHA at new path, reconcile source_path.
Writes: wechat_files, ingestion_log, GCS
2Extract products
Excel / PDF / PPTX / Gemini extractors. Terminal statuses stop re-processing.
Writes: wechat_products, wechat_files.status
2bEnrich category
enrich_categories.classify_batch uses Gemini to assign category + subcategory.
Writes: wechat_products
2cAuto-organize NEW
organize_all(apply=True, move=True, update_firestore=True, only_unorganized=True). Moves new files under Category/Vendor/YYYY-MM-DD layout.
Writes: wechat_files.source_path, .organized_path
3Rebuild vendors
Aggregate wechat_files + wechat_products per vendor: file counts per type, categories, size, last date.
Writes: wechat_vendors
4Status
Timestamped sync record with files_new / products_new / extraction_errors.
Writes: sync_status

File status lifecycle

ingested → vendor_linked or needs_vendor_link → product_extracted | extraction_empty | extraction_failed

The last three are terminal — files in these states are skipped on subsequent sync cycles, preventing the re-extraction loop.
